「Webコンテンツ作成」の成果

本テーマについて

このテーマでは、HTML、CSS、JavaScriptの基本を学び、それらを組み合わせてWebページを構築する方法を習得します。HTMLで構造を、CSSでスタイルを、JavaScriptで動的な動作を制御し、構文リファレンスから必要な情報を得ます。

プロジェクト課題について

このプロジェクトは、選択された都市の天気詳細を表示することを目的としています。HTML、CSS、JavaScriptを組み合わせることで、サーバーからデータを取得できる機能的なウェブサイトを構築しました。ユーザーが都市を選択すると、ウェブサイトはサーバーからデータを取得し、DOM機能を使用してそのデータをウェブサイトに表示します。授業で教えられた要素を統合することで、例えばユーザーがボタンにカーソルを合わせると、11の国とそれぞれの都市のオプションが表示され、任意の都市をクリックすると、その都市に関する天気データが表示されるようになっています。

プロジェクト課題で作成したページ：project.html

数当てゲーム

この数当てゲームでは、HTMLとJavaScriptを使ってシンプルな数当てゲームを作成します。HTMLファイルでは、入力用のテキストボックスと予想を送信するボタンを配置します。JavaScriptファイルでは、1から10までの乱数を「答え」として生成し、ユーザーに最大3回の予想を許可します。各予想は答えと照合され、予想が高すぎるか低すぎるかをフィードバックします。ユーザーが正解を当てるか、全ての試行を使い切るとゲームが終了し、適切なメッセージ,『答えはもっと小さいですよ』や『答えはもっと大きいですよ』などが表示されます。もし、４回以上答えたら、正解の答えと『すでにゲームは終わっています』と表示します。

数当てゲームのページ：kazuate.html

各授業回の学習事項

第１回 WWW と HTML

HTMLでウェブサイトの作り方を学びました。見出しを設定したり、画像やリンクをウェブサイトの中に載せたり、いろいろ試してみました。HTMLでは主にプレーンウェブサイトだけ、文字やハイパーリンクや、画像などを表示します。

演習の成果：練習問題1-4:HTMLのさまざまな要素

第２回 CSS による Web ページの装飾

HTMLで作ったウェブサイトの装飾の設定し方を学びました。色やボックスなどウェブサイトがにぎやかになるために、いろいろ試してみました。混乱を防ぐため、HTMLの中でなく、外部のプログラムを作った方がいいとわかりました。

演習の成果：練習問題2-3: ウェブブラウザ説明ページの装飾

第３回 JavaScript の基本

JavaScriptを使ってページの動きを変えます。例えば、あるオブジェクトからデータを取り出して、WebDeveloperToolsで表示します。そして、コンソールで出力を表示したり、内部のプログラムを確認したりするがJavaScriptあるいはWebDeveloperToolsでやることができます。

演習の成果：練習問題3-6: オブジェクトからデータを取り出す（プロジェクト課題）

第４回 DOM と JavaScript

DOMとはJavaScriptなどを使って、HTMLを書き換えずにウェブサイトの中身を変更します。例えば、HTMLで空っぽの状態で、JavaScriptによって、そのHTMLを書き換え、さらにCSSでの処理もプログラムを読み取れ、ウェブサイトの動きが可能になります。

演習の成果：練習問題4-2から4-5: DOMの練習

第５回 イベント処理

イベント処理で、ウェブサイトに多数の機能をつけて、生きているウェブサイトになるようにしました。ユーザーに答えられることや、入力や選択肢が選べることや、ボタンを押すと何かをやることなど複雑な機能も処理をすることがやってみました。

演習の成果：練習問題5-3: イベントを選択する

コメント

まず、これまでの授業でJavaしか学んでいなかったため、新しいプログラミング言語であるHTMLやJavaScriptに慣れるのに苦労しました。例えば、コードにコメントを追加する方法は、HTMLとJavaScriptでは異なります。このプロジェクトで最も良かった部分は、自分のウェブサイトをJavaScriptを使ってサーバーと接続する方法を学んだことです。これにより、ただのウェブサイトではなく、完全に機能するウェブサイトを作成するための重要なプロセスを習得したように感じます。
もちろん、この学習過程で多くの困難に直面しましたが、それでも非常に良い学習の機会となりました。例えば、HTMLではすべての要素に独自の機能があり、クラスやIDも同様です。例として、a や selectの要素が挙げられますが、option 要素を a 要素内に単純に挿入することはできません。各オプションの値を挿入する方法を見つけるのに苦労しました。
次に、CSSに関しては、作業するのが非常に興味深い部分でした。特に、ボックスの色を変えるなどのhover機能は面白かったです。また、要素の位置を決めたり、ボックスのサイズを設定したり、色を指定したりすることも楽しかったです。しかし、CSSで最も難しい部分は、要素の位置を調整することでした。特に、要素を中央に配置したり、レスポンシブなテーブルを作成したりするのが大変でした。本当に、多くの時間をかけて学び、習得する必要がありましたが、まだ完全にマスターしたとは言えません。
さらに、プロジェクトではSVGマップを使用してみたいとも考えていましたが、非常に時間がかかるため、他の簡単な機能に変更する必要がありました。当初は、カーソルが地図の一部に入ると、その都市の名前が表示され、クリックするとボックスが現れてその都市の天気の詳細が表示されるクリック可能な地図を作成したいと考えていました。また、ボックス内には他の都市に簡単に移動できるドロップダウンメニューを設置する予定でした。しかし、その複雑さから、クラスで学んだすべての機能を適用しながら、期限内に仕上げるために変更する必要がありました。
以上のように、今回のプロジェクトを通じて多くのことを学び、技術的にも成長することができました。新しい言語や技術を学ぶ過程で直面する困難に打ち勝つことができたのは、非常に貴重な経験でした。将来的には、今回の学びを活かしてさらに高度なウェブサイトやアプリケーションを開発していきたいと考えています。
このプロジェクトは、単なる技術の習得だけでなく、問題解決能力や計画力、時間管理の重要性を理解する機会にもなりました。学びの過程で得た知識とスキルを活かし、今後のプロジェクトでも成長を続けていきたいと思います。
